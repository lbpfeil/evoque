<task type="auto">
  <name>Task 1: Install Outfit font and configure CSS</name>
  <files>index.css</files>
  <action>
**Step 1: Install font package**

Run:
```bash
npm install @fontsource-variable/outfit
```

**Step 2: Update index.css**

Add font import at the TOP of index.css (before @tailwind directives):
```css
@import "@fontsource-variable/outfit";
```

Add the --font-sans CSS variable inside :root (after existing variables):
```css
--font-sans: "Outfit Variable", "Outfit", system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
```

The full :root section should include this variable alongside the existing color variables.
  </action>
  <verify>
1. `npm ls @fontsource-variable/outfit` shows package installed
2. `grep "@fontsource-variable/outfit" index.css` finds the import
3. `grep "font-sans.*Outfit" index.css` finds the variable definition
  </verify>
  <done>Outfit font installed and imported in CSS</done>
</task>

<task type="auto">
  <name>Task 2: Update Tailwind config for Outfit font</name>
  <files>tailwind.config.js</files>
  <action>
**Problem:** Current config has:
```javascript
fontFamily: {
  sans: ["var(--font-sans)", ...fontFamily.sans],
}
```

This is correct - it uses the CSS variable. But we should also add Outfit explicitly as a fallback.

**Update tailwind.config.js:**

Change the fontFamily section to:
```javascript
fontFamily: {
  sans: ["Outfit Variable", "Outfit", "var(--font-sans)", ...fontFamily.sans],
  serif: ["Georgia", "Cambria", "Times New Roman", "Times", "serif"],
},
```

This ensures:
1. "Outfit Variable" is tried first (the variable font)
2. "Outfit" as fallback (if variable font fails)
3. CSS variable as another fallback
4. System fonts as final fallback
5. Also define serif for the study cards (they use `font-serif` class)
  </action>
  <verify>
1. Run `npm run dev` and open the app
2. Check DevTools > Computed styles on any text element
3. Font-family should show "Outfit Variable" or "Outfit"
4. Text should render cleanly without appearing strange
  </verify>
  <done>Tailwind configured with Outfit font family</done>
</task>
